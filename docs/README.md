# Documentation Map

This repository keeps auto-discovered agent runtime files at the repo root:

- `AGENTS.md` for Codex and similar repo-aware agents
- `CLAUDE.md` for Claude Code and similar chat-first agents

Those files stay at the root intentionally so agent tools can discover them automatically.

Everything else is grouped by purpose:

- `docs/ARCHITECTURE.md` — deep technical reference, data flow, extension notes
- `docs/MASTER_GAP_MATRIX.md` — master-program audit of current main vs the next-level consultant contract
- `docs/ARTIFACT_GUIDE.md` — what the emitted artifacts are for and who should read them
- `docs/ARTIFACT_VERSIONING_POLICY.md` — compatibility and schema-version policy
- `docs/AGENT_CONTROL_PLANE.md` — role registry, handoffs, budgets, and stop-condition contract
- `docs/GOVERNANCE_AND_SOURCE_POLICY.md` — source-use, confidence, and input-handling hygiene
- `docs/MASTER_ACCEPTANCE_CHECKLIST.md` — release gate for the consultant contract
- `docs/GRAPHICS_AGENT.md` — chart-selection and styling guide for the graphics agent
- `.github/CONTRIBUTING.md` — contributor workflow
- `.github/SECURITY.md` — security policy
- `.github/CODE_OF_CONDUCT.md` — community expectations
